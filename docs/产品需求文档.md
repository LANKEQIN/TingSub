# 汀阅（TingSub）产品需求文档

## 一、产品概述

### 1.1 应用定位
汀阅（TingSub）是一款专为订阅经济时代打造的一站式订阅管理应用，旨在帮助用户集中管理各类线上线下订阅服务，解决订阅分散、遗忘到期、难以统计等痛点问题。

### 1.2 核心价值
- **一站式管理**：将所有订阅服务集中在一个平台，告别分散管理的烦恼
- **智能提醒**：多渠道到期提醒，避免不必要的自动续费损失
- **消费分析**：可视化统计订阅支出，帮助用户优化订阅策略
- **数据安全**：本地加密存储，保护用户隐私数据

### 1.3 应用名称
- 中文名：汀阅
- 英文名：TingSub

## 二、目标用户

### 2.1 核心用户群体
1. **高频订阅用户**：拥有5个以上各类线上/线下订阅服务的用户
2. **消费敏感型用户**：关注订阅成本、希望优化订阅支出的用户
3. **健忘型用户**：经常错过订阅到期时间导致自动续费的用户
4. **多平台用户**：同时使用iOS和Android设备的跨平台用户

### 2.2 用户画像
- **年龄范围**：18-45岁
- **职业分布**：白领、学生、自由职业者等
- **消费习惯**：愿意为优质服务付费，但关注成本效益
- **技术能力**：具备基本智能手机操作能力

## 三、用户故事与使用场景

### 3.1 用户故事

#### 3.1.1 新用户首次使用
**作为**一名新用户
**我想要**快速添加我的订阅服务
**以便于**在一个地方管理所有订阅

**验收标准**：
- 应用启动后显示引导页面
- 提供常见订阅服务模板（如Netflix、Spotify等）
- 支持手动输入订阅信息
- 首次添加订阅后显示成功提示

#### 3.1.2 订阅到期提醒
**作为**一名订阅用户
**我想要**在订阅到期前收到提醒
**以便于**决定是否续费或取消订阅

**验收标准**：
- 支持设置提前1-7天提醒
- 提醒消息包含订阅名称、到期日期、费用
- 支持点击提醒直接跳转到订阅详情
- 支持关闭单个订阅的提醒

#### 3.1.3 查看订阅统计
**作为**一名消费敏感型用户
**我想要**查看我的订阅支出统计
**以便于**了解我的订阅消费情况并优化支出

**验收标准**：
- 显示月度/年度总支出
- 按分类展示支出占比
- 支持查看支出趋势图
- 支持导出统计数据

#### 3.1.4 分类管理订阅
**作为**一名拥有多个订阅的用户
**我想要**按分类管理我的订阅
**以便于**更清晰地查看不同类型的订阅

**验收标准**：
- 支持创建自定义分类
- 支持为订阅分配分类
- 支持按分类筛选订阅
- 显示每个分类的订阅数量和支出

#### 3.1.5 数据导出与备份
**作为**一名注重数据安全的用户
**我想要**导出我的订阅数据
**以便于**备份或在其他设备上使用

**验收标准**：
- 支持导出为CSV/JSON格式
- 支持选择导出全部或部分数据
- 导出文件包含完整的订阅信息
- 支持导入导出的数据

### 3.2 使用场景

#### 3.2.1 场景一：新用户首次使用
**背景**：小王是一名刚毕业的大学生，拥有Netflix、Spotify、Adobe Creative Cloud等多个订阅服务，但经常忘记到期时间。

**操作流程**：
1. 下载并安装汀阅应用
2. 完成引导页面设置（选择主题、货币等）
3. 点击"添加订阅"按钮
4. 从预设模板中选择Netflix，填写费用和到期日期
5. 重复步骤添加其他订阅
6. 设置提前3天到期提醒
7. 查看订阅列表，确认所有订阅已添加

**预期结果**：小王成功添加了所有订阅，并设置了到期提醒，不再担心忘记续费。

#### 3.2.2 场景二：订阅到期提醒
**背景**：小李的Netflix订阅将在3天后到期，他需要决定是否续费。

**操作流程**：
1. 收到汀阅应用的推送通知："Netflix将在3天后到期"
2. 点击通知进入订阅详情页面
3. 查看订阅历史和费用信息
4. 决定续费，更新到期日期
5. 或决定取消，将订阅状态改为"已取消"

**预期结果**：小李及时处理了订阅到期问题，避免了不必要的自动续费。

#### 3.2.3 场景三：查看消费统计
**背景**：小张想了解自己每月的订阅支出情况，优化订阅策略。

**操作流程**：
1. 打开汀阅应用，进入"统计"页面
2. 查看月度总支出：¥298
3. 查看分类占比：视频60%、音乐25%、工具15%
4. 查看支出趋势图，发现最近3个月支出稳定
5. 导出统计数据为CSV格式

**预期结果**：小张清楚了解了自己的订阅支出情况，可以做出更明智的消费决策。

#### 3.2.4 场景四：分类管理订阅
**背景**：小陈拥有20多个订阅服务，想要按类型管理。

**操作流程**：
1. 进入"分类"页面，查看预设分类
2. 创建自定义分类"云存储"
3. 将iCloud、Google Drive等订阅分配到"云存储"分类
4. 在订阅列表中按分类筛选，查看"云存储"分类下的订阅
5. 查看每个分类的订阅数量和支出

**预期结果**：小陈的订阅按类型清晰分类，管理更加便捷。

#### 3.2.5 场景五：数据导出与备份
**背景**：小林要换新手机，需要备份订阅数据。

**操作流程**：
1. 进入"设置"页面
2. 点击"数据导出"选项
3. 选择导出格式为JSON
4. 选择导出全部数据
5. 保存导出文件到本地存储
6. 在新手机上安装汀阅应用
7. 点击"数据导入"，选择之前导出的文件
8. 确认导入成功

**预期结果**：小林成功在新手机上恢复了所有订阅数据。

## 四、核心功能需求

### 4.1 订阅信息管理
#### 4.1.1 订阅添加
- 手动录入：支持用户手动输入订阅名称、费用、周期等信息
- 自动识别：通过AI技术自动识别短信、邮件中的订阅信息（后续版本迭代）
- 模板导入：提供常见订阅服务模板，快速添加

#### 4.1.2 订阅编辑
- 支持修改订阅的所有信息字段
- 批量操作：支持批量修改订阅分类、标签等

#### 4.1.3 订阅删除
- 支持单个订阅删除
- 批量删除：支持批量删除选中的订阅
- 删除确认：删除前二次确认，防止误操作

#### 4.1.4 订阅查看
- 列表视图：以列表形式展示所有订阅服务
- 卡片视图：以卡片形式展示订阅关键信息
- 详情页面：展示订阅的完整信息和操作选项

### 4.2 分类与标签
#### 4.2.1 分类管理
- 预设分类：提供视频、音乐、工具、教育、生活等预设分类
- 自定义分类：支持用户创建、编辑、删除自定义分类
- 分类统计：按分类统计订阅数量和支出

#### 4.2.2 标签体系
- 自由标签：支持用户为订阅添加任意标签
- 标签筛选：支持按标签筛选订阅服务
- 标签云：展示用户常用标签

### 4.3 到期提醒
#### 4.3.1 提醒设置
- 提前提醒：支持设置提前1-7天提醒
- 提醒渠道：支持本地推送通知
- 重复提醒：支持设置到期日当天多次提醒

#### 4.3.2 提醒管理
- 提醒历史：查看所有提醒记录
- 提醒开关：全局或单个订阅的提醒开关

### 4.4 统计分析
#### 4.4.1 支出统计
- 月度统计：统计当月订阅支出
- 年度统计：统计全年订阅支出
- 趋势分析：展示订阅支出的月度/年度变化趋势

#### 4.4.2 分类分析
- 分类占比：展示各分类订阅支出占比
- 订阅数量：统计各分类订阅数量

#### 4.4.3 订阅洞察
- 订阅推荐：根据用户订阅习惯推荐优质服务
- 浪费分析：识别长期未使用的订阅服务

### 4.5 数据导出
- 导出格式：支持导出为CSV/Excel格式
- 导出内容：支持选择导出全部或部分订阅数据
- 导出路径：支持选择导出到本地存储或云端

## 五、竞品分析与差异化优势

### 5.1 竞品分析

#### 5.1.1 Bobby
**产品定位**：订阅管理应用，支持iOS和Android平台
**核心功能**：
- 订阅信息管理
- 到期提醒
- 支出统计
- 多货币支持

**优势**：
- 界面设计精美
- 支持多货币
- 提供订阅模板

**劣势**：
- 需要付费订阅才能使用完整功能
- 数据存储在云端，隐私担忧
- 分类管理功能较弱

#### 5.1.2 Subscript
**产品定位**：轻量级订阅管理工具
**核心功能**：
- 订阅信息管理
- 到期提醒
- 简单统计

**优势**：
- 完全免费
- 界面简洁
- 轻量级应用

**劣势**：
- 功能较为基础
- 缺乏深度统计分析
- 不支持数据导出

#### 5.1.3 Truebill
**产品定位**：订阅管理+财务分析应用
**核心功能**：
- 订阅信息管理
- 到期提醒
- 支出统计
- 订阅取消服务

**优势**：
- 功能全面
- 提供订阅取消服务
- 支持银行账户连接

**劣势**：
- 仅支持美国市场
- 需要连接银行账户
- 隐私风险较高

### 5.2 汀阅的差异化优势

#### 5.2.1 本地优先的数据安全
**优势描述**：汀阅采用本地加密存储，用户数据完全掌控在自己手中，无需担心云端数据泄露。
**用户价值**：
- 隐私保护：数据不上传云端，避免隐私泄露风险
- 数据主权：用户完全掌控自己的数据
- 离线可用：无需网络连接即可使用核心功能

#### 5.2.2 跨平台一致性体验
**优势描述**：汀阅基于Expo React Native开发，确保iOS和Android平台功能一致，体验统一。
**用户价值**：
- 一致体验：跨平台功能完全一致，无功能差异
- 数据同步：支持数据导出导入，跨设备数据迁移
- 统一设计：遵循Material Design规范，界面美观统一

#### 5.2.3 智能化提醒系统
**优势描述**：汀阅提供灵活的提醒设置，支持提前1-7天提醒、重复提醒、多渠道通知等。
**用户价值**：
- 灵活设置：根据个人需求自定义提醒时间
- 多渠道通知：支持本地推送、震动、声音等多种提醒方式
- 提醒历史：记录所有提醒，方便查看历史

#### 5.2.4 深度统计分析
**优势描述**：汀阅提供多维度的统计分析，包括支出趋势、分类占比、订阅洞察等。
**用户价值**：
- 全面统计：月度、年度、趋势分析一应俱全
- 可视化展示：图表直观展示数据
- 消费洞察：帮助用户优化订阅策略

#### 5.2.5 开源透明
**优势描述**：汀阅计划开源，代码透明，用户可以自行审查和修改。
**用户价值**：
- 代码透明：用户可以审查代码，确保无后门
- 社区贡献：用户可以参与功能开发
- 自定义扩展：技术用户可以自行扩展功能

## 六、非功能需求

### 6.1 性能需求
- 应用启动时间：<2秒
- 页面切换时间：<500毫秒
- 数据加载时间：<1秒（本地数据）

### 6.2 兼容性需求
- iOS：支持iOS 14及以上版本
- Android：支持Android 10及以上版本
- 设备：支持手机、平板等多种设备

### 6.3 安全性需求
- 数据加密：本地数据采用AES-256加密存储
- 隐私保护：严格遵守GDPR、CCPA等隐私法规
- 权限管理：仅申请必要的系统权限

### 6.4 可用性需求
- 界面设计：遵循Material Design规范，简洁直观
- 操作流程：核心功能操作步骤<3步
- 错误处理：友好的错误提示和恢复机制

### 6.5 可维护性需求
- 代码规范：遵循ESLint、Prettier等代码规范
- 文档完备：提供完整的开发文档和API文档
- 测试覆盖：单元测试覆盖率>80%
