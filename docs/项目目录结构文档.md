# 汀阅（TingSub）项目目录结构文档

## 一、目录结构概述

### 1.1 设计原则
汀阅（TingSub）采用模块化、分层化的目录结构设计，确保代码组织清晰、易于维护和扩展。

### 1.2 核心设计原则
1. **模块化**：按功能模块组织代码，降低耦合度
2. **分层清晰**：UI层、业务逻辑层、数据层职责明确
3. **易于查找**：文件命名规范，目录结构直观
4. **可扩展性**：预留扩展空间，便于功能迭代
5. **一致性**：遵循统一的命名和组织规范

## 四、文件命名规范

### 4.1 组件文件命名
- **组件文件**：使用帕斯卡命名法（PascalCase）
- **示例**：`Button.tsx`、`SubscriptionCard.tsx`、`HomeScreen.tsx`

### 4.2 工具文件命名
- **工具文件**：使用驼峰命名法（camelCase）
- **示例**：`dateUtils.ts`、`validationUtils.ts`、`formatUtils.ts`

### 4.3 类型文件命名
- **类型文件**：使用驼峰命名法（camelCase）
- **示例**：`user.ts`、`subscription.ts`、`category.ts`

### 4.4 常量文件命名
- **常量文件**：使用驼峰命名法（camelCase）
- **示例**：`appConstants.ts`、`storageKeys.ts`、`theme.ts`

### 4.5 配置文件命名
- **配置文件**：使用驼峰命名法（camelCase）
- **示例**：`env.ts`、`realm.ts`

### 4.6 测试文件命名
- **测试文件**：与源文件同名，添加`.test`后缀
- **示例**：`Button.test.tsx`、`dateUtils.test.ts`

### 4.7 样式文件命名
- **样式文件**：与组件同名，添加`.styles`后缀
- **示例**：`Button.styles.ts`、`HomeScreen.styles.ts`

## 五、模块组织原则

### 5.1 按功能模块组织
- **原则**：相关功能的代码放在同一目录下
- **示例**：订阅相关组件放在`components/subscription/`目录下

### 5.2 按层次组织
- **原则**：UI层、业务逻辑层、数据层分层组织
- **UI层**：`components/`、`screens/`
- **业务逻辑层**：`services/`、`hooks/`
- **数据层**：`repositories/`、`models/`

### 5.3 按类型组织
- **原则**：相同类型的文件放在同一目录下
- **示例**：所有工具函数放在`utils/`目录下

### 5.4 按平台组织
- **原则**：平台特定的代码放在对应平台目录下
- **示例**：Android代码放在`android/`目录下，iOS代码放在`ios/`目录下

## 六、代码分层架构

### 6.1 UI层
**职责**：负责用户界面展示和用户交互
**目录**：`components/`、`screens/`
**特点**：
- 纯展示逻辑，不包含业务逻辑
- 通过Hooks获取数据和操作
- 使用组件库构建UI

### 6.2 业务逻辑层
**职责**：负责业务逻辑处理和状态管理
**目录**：`services/`、`hooks/`、`store/`
**特点**：
- 封装业务逻辑
- 管理应用状态
- 协调数据访问

### 6.3 数据访问层
**职责**：负责数据持久化和数据访问
**目录**：`repositories/`、`models/`
**特点**：
- 封装数据访问逻辑
- 提供统一的数据接口
- 处理数据验证和转换

### 6.4 基础设施层
**职责**：提供基础功能和工具
**目录**：`utils/`、`constants/`、`config/`、`types/`
**特点**：
- 提供通用工具函数
- 定义应用常量和配置
- 定义类型和接口

## 七、目录结构最佳实践

### 7.1 保持目录结构简洁
- 避免过深的目录嵌套
- 目录层级不超过3层
- 相关文件放在同一目录下

### 7.2 遵循命名规范
- 使用一致的命名规范
- 文件名清晰表达文件用途
- 避免使用缩写和拼音

### 7.3 合理组织代码
- 按功能模块组织代码
- 按层次组织代码
- 按类型组织代码

### 7.4 保持文件职责单一
- 每个文件只负责一个功能
- 避免文件过大（不超过2000行）
- 复杂组件拆分为子组件

### 7.5 使用索引文件
- 在目录中使用`index.ts`统一导出
- 简化导入路径
- 提高代码可维护性
